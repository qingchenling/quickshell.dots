This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

A [Quickshell](https://quickshell.org/) desktop shell configuration for Wayland/Hyprland — a full panel, app launcher, notification center, settings center, and animated wallpaper background, all written in QML.

- **License**: GPLv3
- **Entry point**: `shell.qml` — instantiates `ShellRoot` with four top-level widgets
- **No build step**: QML files are interpreted directly by `quickshell`

## How to run

```bash
quickshell
```

Quickshell must be running as a Wayland client (typically launched from Hyprland config). There are no tests, linters, or build tools in this project.

### Key Quickshell concepts used

- **`PanelWindow`**: Docks to a screen edge; used for the panel, app launcher, notifications, and background.
- **`PopupWindow`**: Floats anchored to a panel window; use `grabFocus: true` when keyboard input is needed. Used for start menu, key tips, calendar, dynamic island, network picker.
- **`FloatingWindow`**: Regular toplevel OS window, positioned by the compositor. Set `screen: panel.screen` to appear on the same output. Used for the settings center.
- **`ShellRoot`**: Root object that manages all windows.
- **`FileView`**: Reactive file reader — used to read SVG contents (for colour replacement), sysfs LED state files, and `settings.json`.
- **`Process`**: Executes shell commands (`poweroff`, `reboot`, `brightnessctl`, `matugen`).
- **`IpcHandler`**: Enables toggling windows via IPC (e.g., `AppLauncher.toggle()`).
- **`pragma Singleton`**: QML singleton pattern — services and Colors are singletons imported as `import qs.Services` / `import qs.Themes`.
- **`ClippingRectangle`** (`Quickshell.Widgets`): Shader-based rounded-corner clipping that actually works — prefer over `Rectangle { clip: true }` when clipping images.

### Colour/theme system

- `Colors.qml` (singleton, `import qs.Themes`) loads `colors.json` (generated by [matugen](https://github.com/InioX/matugen)) and exposes 51 MD3 token properties directly (e.g. `Colors.surface`, `Colors.primary`, `Colors.on_surface`, `Colors.secondary_container`, `Colors.outline_variant`, etc.).
- `colors.json` follows Material Design 3 token naming (`surface`, `primary`, `secondary`, `tertiary`, `surface_variant`, `outline`, etc.)
- **Reactivity**: `FileView.onFileChanged` triggers `reload()`, which replaces the entire data object reference. QML's binding engine tracks every `color: Colors.surface` binding and re-evaluates automatically when `colors.json` is rewritten.
- When the wallpaper changes, `Background.qml` runs `matugen image <path>` which regenerates `colors.json`, and all themed components pick up the new colours without manual intervention.
- Always access colour properties directly on the Colors singleton (e.g. `Colors.surface`, `Colors.on_surface`).

## Components catalog

All components live in `Components/` and are imported via `import qs.Components`.
Components must NOT import services directly — receive data through properties and emit signals for actions.

### Primitive components

| Component | Properties | Signals | Notes |
|---|---|---|---|
| **`IconSvg`** | `path: url`, `color: color` | — | Loads SVG, replaces fill/stroke at runtime via FileView |
| **`Button`** | `text`, `activeText`, `icon`, `activeIcon`, `active`, `bgColor`, `fgColor`, `activeBgColor`, `activeFgColor`, `hovered` | `clicked()` | Themed button with icon+text, hover/active states, press-scale (0.92) |
| **`TextField`** | `bgColor`, `fgColor` | — | Themed text input with rounded background |
| **`Slide`** | `icon`, `value`, `minn`, `maxn`, `delta` | `deltaChanged()` | Slider with wheel/tap — used for volume + brightness |
| **`ToggleSwitch`** | `checked: bool` | `toggled()` | Material You toggle with animated thumb knob |
| **`FadeTip`** | — | — | Button variant with fade-in/pause/fade-out animation cycle |

### Layout / structural components

| Component | Properties | Notes |
|---|---|---|
| **`SectionHeader`** | `text: string` | Section title — 12px bold, `Colors.on_surface_variant`, 32px height. Use for every section label in settings pages. |
| **`CollapsibleSection`** | `expanded: bool`, `heightWhenVisible: real` | Wraps content with height + opacity animation (250ms OutCubic). Use instead of manually wiring `Behavior on height` + `clip: true`. |
| **`PageHeader`** | `title: string`, `showToggle: bool`, `toggleChecked: bool` | `toggled()` | Page-top title bar (20px bold). Optionally shows a ToggleSwitch on the right. Use for settings page headers. |
| **`PillGroup`** | `model: array[{label, value}]`, `currentValue: var` | `selected(value)` | Selectable pill buttons in a Flow. Selected pill gets `primary_container` background. Use for option selectors. |

### Content components

| Component | Properties | Signals | Notes |
|---|---|---|---|
| **`WifiNetworkCard`** | `network`, `selected`, `showPassword`, `passwordText`, `signalLevel`, `secured`, `iconPath` | `tapped(network)`, `submitPassword()` | WiFi network row with expandable inline password field |

### When to use each

```
Section "Label"        →  SectionHeader { text: "Label" }
Conditional region     →  CollapsibleSection { expanded: cond; heightWhenVisible: N }
Page top title+toggle  →  PageHeader { title: "X"; showToggle: true; onToggled: ... }
Pick one from options  →  PillGroup { model: [...]; currentValue: sel; onSelected: ... }
WiFi network row       →  WifiNetworkCard { network: net; ... }
```

**Anti-patterns to avoid:**
- Don't inline `Behavior on height` + `clip: true` — use `CollapsibleSection`
- Don't repeat the section-title Text pattern — use `SectionHeader`
- Don't use `Rectangle { clip: true }` for clipping images — use `ClippingRectangle` from `Quickshell.Widgets`
- Don't duplicate the page title + toggle layout — use `PageHeader`

### Data flow: wallpaper change

1. User clicks "next wallpaper" button in StartMenu → `BackgroundService.changeImage("")`
2. `BackgroundService` picks a random image from `~/Pictures/Wallpapers/`
3. `Background.qml` receives the `onChangeImage` signal, calls `pickShader()` to read `SettingsService.shader`, starts shader transition animation
4. Simultaneously runs `matugen image <file>` to regenerate `colors.json`
5. `FileView` in `Colors.qml` detects the file change and reloads, updating all bound colours

### Data flow: settings persistence

1. User changes a setting in the UI → writes to `SettingsService` property (e.g. `SettingsService.wallpaper = path`)
2. `SettingsService` debounces (300ms) and writes `settings.json` via `Process`
3. On next startup, `FileView` reads `settings.json` → `_loadFromJson()` restores all properties
4. `_loading` flag prevents write-back loops during load

### External integrations

- **Hyprland**: workspace list, focused workspace toplevel count (blur toggle)
- **Pipewire**: default audio sink volume control
- **UPower**: battery state, percentage, time remaining
- **SystemTray**: SNI tray icons with left/right click support
- **Notifications**: Freedesktop notification server via `NotificationServer`
- **matugen**: Material You colour generation from wallpaper images
- **brightnessctl**: Backlight brightness control

### Frontend/backend separation

- **Services/** (backend): QML singletons wrapping Quickshell/system APIs. Own all business logic — network scanning/sorting, wallpaper management, idle inhibition, settings persistence.
- **Widgets/** (frontend): Pure UI — layout, colours, animations. Import services via `import qs.Services`. Keep UI state (selection, input) locally.
- **Components/** (shared): Reusable UI primitives. No direct service imports — receive data through properties and emit signals for actions.

### Settings Center architecture

```
Panel.qml
├── StartMenu.qml  →  [⚙ gear button] → panel.toggleSettings()
└── SettingsCenter.qml  (FloatingWindow on panel.screen)
    ├── Tab bar (left, 76px) — icon + label per tab, selected indicator
    └── Page Loader (right) — slide+fade MD3 transitions
        ├── WifiPage.qml      ←  NetworkService, WifiNetworkCard
        ├── BluetoothPage.qml  ←  local state (placeholder backend)
        └── WallpaperPage.qml  ←  SettingsService, BackgroundService
```

To add a new settings page:
1. Create `Widgets/Settings/pages/YourPage.qml`
2. Add a `ListElement { name: "..."; icon: "..."; page: "YourPage.qml" }` to `tabs` in `SettingsCenter.qml`
3. Use `PageHeader`, `SectionHeader`, `CollapsibleSection` from Components for layout

### Adding new settings to persistence

1. Add a `property type name: default` to `SettingsService.qml`
2. Add `onNameChanged: { if (!_loading) scheduleSave(); }`
3. Add it to `_loadFromJson()` (read) and `_writeSettings()` (write)
