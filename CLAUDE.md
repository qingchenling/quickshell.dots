## Project overview

A [Quickshell](https://quickshell.org/) desktop shell configuration for Wayland/Hyprland — a full panel, app launcher, notification center, and animated wallpaper background, all written in QML.

- **License**: GPLv3
- **Entry point**: `shell.qml` — instantiates `ShellRoot` with four top-level widgets
- **No build step**: QML files are interpreted directly by `quickshell`

## How to run

```bash
quickshell
```

Quickshell must be running as a Wayland client (typically launched from Hyprland config). There are no tests, linters, or build tools in this project.

## Architecture

### Top-level widgets (`shell.qml`)

| Widget | Purpose |
|---|---|
| `Panel` | Top bar with start menu, workspaces, dynamic island, key indicators, system tray, battery, network, clock |
| `AppLauncher` | Bottom-anchored popup with searchable app list (pinyin-aware) |
| `NotificationManager` | Right-side notification panel via `NotificationServer` |
| `Background` | Full-screen wallpaper layer with shader transitions + matugen color extraction |

### Key Quickshell concepts used

- **`PanelWindow`**: Docks to a screen edge; used for the panel, app launcher, notifications, and background.
- **`PopupWindow`**: Floats anchored to a panel window; use `grabFocus: true` when keyboard input is needed. Used for start menu, key tips, calendar, dynamic island, network picker.
- **`ShellRoot`**: Root object that manages all windows.
- **`FileView`**: Reactive file reader — used to read SVG contents (for colour replacement) and sysfs LED state files.
- **`Process`**: Executes shell commands (`poweroff`, `reboot`, `brightnessctl`, `matugen`).
- **`IpcHandler`**: Enables toggling windows via IPC (e.g., `AppLauncher.toggle()`).
- **`pragma Singleton`**: QML singleton pattern — `Colors`, `BackgroundService`, `IdleService`, `NetworkService` are singletons imported as `import qs.Services` / `import qs.Themes`.

### Colour/theme system

- `Colors.qml` (singleton, `import qs.Themes`) loads `colors.json` (generated by [matugen](https://github.com/InioX/matugen)) and exposes 51 MD3 token properties directly (e.g. `Colors.surface`, `Colors.primary`, `Colors.on_surface`, `Colors.secondary_container`, `Colors.outline_variant`, etc.).
- `colors.json` follows Material Design 3 token naming (`surface`, `primary`, `secondary`, `tertiary`, `surface_variant`, `outline`, etc.)
- **Reactivity**: `FileView.onFileChanged` triggers `reload()`, which replaces the entire data object reference. QML's binding engine tracks every `color: Colors.surface` binding and re-evaluates automatically when `colors.json` is rewritten.
- When the wallpaper changes, `Background.qml` runs `matugen image <path>` which regenerates `colors.json`, and all themed components pick up the new colours without manual intervention.
- Always access colour properties directly on the Colors singleton (e.g. `Colors.surface`, `Colors.on_surface`).

### Data flow: wallpaper change

1. User clicks "next wallpaper" button in StartMenu → `BackgroundService.changeImage("")`
2. `BackgroundService` picks a random image from `~/Pictures/Wallpapers/`
3. `Background.qml` receives the `onChangeImage` signal, starts shader transition animation
4. Simultaneously runs `matugen image <file>` to regenerate `colors.json`
5. `FileView` in `Colors.qml` detects the file change and reloads, updating all bound colours

### External integrations

- **Hyprland**: workspace list, focused workspace toplevel count (blur toggle)
- **Pipewire**: default audio sink volume control
- **UPower**: battery state, percentage, time remaining
- **SystemTray**: SNI tray icons with left/right click support
- **Notifications**: Freedesktop notification server via `NotificationServer`
- **matugen**: Material You colour generation from wallpaper images
- **brightnessctl**: Backlight brightness control

### Frontend/backend separation

- **Services/** (backend): QML singletons wrapping Quickshell/system APIs. Own all business logic — network scanning/sorting, wallpaper management, idle inhibition.
- **Widgets/** (frontend): Pure UI — layout, colours, animations. Import services via `import qs.Services`. Keep UI state (selection, input) locally.
- **Components/** (shared): Reusable UI primitives. No direct service imports — receive data through properties and emit signals for actions.
